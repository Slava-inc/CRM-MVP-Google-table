# CRM MVP в Google Таблицах с интеграцией GPT (ручной)

## Цель системы

Создать простую CRM в Google Таблицах для персонального ведения клиентов, отслеживания этапов касаний, работы по системе «Стадия + Состояние → Касание» и ручного использования GPT для персонализации и анализа.

## Структура системы (вкладки в таблице)

### 1. `Клиенты`

**Цель:** Хранение основной информации о каждом клиенте, его стадии, состоянии, касаниях.

| Колонка | Название поля | Тип данных | Описание |
|---------|---------------|------------|----------|
| A | `ID` | Число | Уникальный ID клиента (1, 2, 3 и т.д.). |
| B | `ФИО` | Текст | Имя клиента. |
| C | `Контакты` | Текст | Telegram / Email / Телефон. |
| D | `Дата первого контакта` | Дата | Формат: ДД.ММ.ГГГГ. |
| E | `Текущий статус/ Этап воронки` | Выпадающий список | `0 касаний`, `1 касание`, ..., `8 касаний`. |
| F | `Стадия` | Выпадающий список | `хаос`, `идентификация`, `продукт`, `рост`. |
| G | `Эмоциональное состояние` | Выпадающий список | `усталость`, `сомнение`, `страх`, `мотивация`, `равнодушие`. |
| H | `Последнее касание` | Текст | Дата + формат (например, "2025-01-15, звонок"). |
| I | `Пробои` | Текст | Краткое описание выявленных слабых мест. |
| J | `Назначенные материалы/гайды` | Текст | Названия, которые были отправлены (через запятую). |
| K | `Примечания` | Текст | Любая важная информация из переписки. |

---

### 2. `История касаний`

**Цель:** Лог всех взаимодействий с клиентами.

| Колонка | Название поля | Тип данных | Описание |
|---------|---------------|------------|----------|
| A | `ID строки` | Число | Уникальный ID строки (1, 2, 3...). |
| B | `Дата касания` | Дата | Формат: ДД.ММ.ГГГГ. |
| C | `ID клиента` | Число | Связь с `Клиенты::ID`. |
| D | `Тип касания` | Выпадающий список | `звонок`, `письмо`, `сообщение`, `PDF`, `аудио`, `GPT`, `встреча`. |
| E | `Инструмент или материал` | Текст | Название гайда, файла, инструмента. |
| F | `Результат` | Текст | Реакция клиента, кратко. |
| G | `План следующего шага` | Текст | Что делать дальше. |
| H | `Ответственный` | Выпадающий список | `МОП`, `владелец`, `РОП`. |

---

### 3. `Гайды/Материалы`

**Цель:** База знаний, содержащая все материалы и гайды, которые можно использовать с клиентами.

| Колонка | Название поля | Тип данных | Описание |
|---------|---------------|------------|----------|
| A | `ID` | Число | Уникальный ID материала. |
| B | `Название` | Текст | Например, "Гайд по автоматизации". |
| C | `Формат` | Выпадающий список | `PDF`, `аудио`, `GPT-подсказка`, `голосовое`, `видео`. |
| D | `Назначение` | Текст | Что решает: "повышение мотивации", "анализ ошибок". |
| E | `Для стадии` | Выпадающий список | `хаос`, `идентификация`, `продукт`, `рост`. |
| F | `Для эмоционального состояния` | Выпадающий список | `усталость`, `сомнение`, `страх`, `мотивация`. |
| G | `Краткое описание` | Текст | 1-2 предложения. |
| H | `Примеры использования` | Текст | Примеры кейсов. |


---

### 4. `GPT-ассистент`

**Цель:** Инструкции и шаблоны для работы с GPT вручную.

| Колонка | Название поля | Тип данных | Описание |
|---------|---------------|------------|----------|
| A | `Назначение` | Текст | `Анализ переписки`, `Подбор инсайтов`, `Генерация сообщений`. |
| B | `Промпт-шаблон` | Текст | Готовый шаблон, который можно вставить в GPT. |
| C | `Пример` | Текст | Пример заполнения. |
| D | `Инструкция` | Текст | Как использовать. |

**Пример содержимого:**

| Назначение | Промпт-шаблон | Пример | Инструкция |
|------------|---------------|--------|------------|
| Генерация сообщения | Привет, {Имя}! На стадии "{Стадия}" и с эмоцией "{Эмоция}", вам может быть полезна наша новая презентация: {Название}. | Привет, Иван! На стадии "продукт" и с эмоцией "сомнение"... | Вставьте профиль клиента и описание презентации в шаблон. |
| Анализ переписки | Проанализируй переписку: {ТЕКСТ ПЕРЕПИСКИ}. Найди эмоции, потребности, пробои. | ... | Скопируйте переписку и вставьте сюда. |


---

### 5. `Анализ диалогов` (опционально)

**Цель:** Ручной анализ переписок с клиентами с помощью GPT.

| Колонка | Название поля | Тип данных | Описание |
|---------|---------------|------------|----------|
| A | `ID строки` | Число | Уникальный ID. |
| B | `ID клиента` | Число | Ссылка на `Клиенты::ID`. |
| C | `Диалог` | Текст | Вставленный текст переписки. |
| D | `GPT-анализ` | Текст | Вставленный ответ GPT. |
| E | `Рекомендации` | Текст | Что сделать дальше. |
